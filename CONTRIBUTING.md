# 贡献指南

状态：当前公开仓库说明。

## 贡献范围

Noema 早期公开阶段优先接受：

- 可复现的 bug report。
- 能稳定失败和通过的测试用例。
- 文档修正。
- 构建、发布、权限、隐私和网络请求相关问题。
- 小范围 UI 文案和可访问性改进。

以下改动需要先开 issue 讨论，不建议直接提交大型 PR：

- 改变照片读取、存储、删除或上传边界。
- 改变 AI Provider 默认行为。
- 新增默认联网能力、权限、SDK 或外部服务。
- 大规模重构、平台范围变化或新的商业化能力。

## 本地开发

```bash
flutter pub get
flutter test
flutter analyze
```

提交 PR 前请说明：

- 改动解决什么问题。
- 是否改变照片读取、存储、AI 请求或网络行为。
- 是否新增依赖、权限或外部服务。
- 是否包含截图或测试结果。

## 隐私边界

不要提交：

- 真实 API Key、token、证书或 keystore。
- 真实用户照片。
- 未脱敏日志。
- 含本地绝对路径、设备序列号、账号信息的截图或报告。
- 来源不清晰的图片、字体或设计素材。

## 代码风格

遵循现有 Flutter/Dart 项目结构。能复用现有组件和状态模型时，不新增平行抽象。涉及行为变化时优先补测试。
