# 隐私说明

状态：当前公开仓库说明。

## 简短版

Noema 的默认设计是本地优先：用户主动选择照片后，App 在设备上建立整理空间、预览、分组、评审和浏览流程。Noema 不默认后台扫描全图库，不自动删除照片，也不默认把照片上传到作者服务器。

## 照片

- 用户主动选择或导入的照片才进入 Noema 工作流。
- Noema 会在本机使用照片元数据、缩略图、预览图和评审结果来支持整理体验。
- Noema 不会在没有用户动作的情况下默认上传照片。

## AI 品鉴

`鉴` 中的 AI 品鉴是可选能力。只有在用户配置 Provider、Base URL、Model 和 API Key，并主动触发品鉴时，相关照片数据才会发送给用户选择的 Provider。

第三方 Provider 如何保存、训练、审计或转发数据，由该 Provider 的服务条款和隐私政策决定。

## API Key

API Key 保存在设备安全存储中。Noema UI 不回显已保存的 API Key；重新输入会替换旧密钥。公开仓库和官方 release 不应包含任何默认密钥、测试密钥或真实用户密钥。

## 作者服务器

当前公开准备方案不包含作者服务器。AI 请求不经过作者服务器。

## 发布前验证

公开 release 前需要验证：

- Android 权限。
- 网络请求目的地。
- 日志脱敏。
- APK SHA256。
- 素材授权。
