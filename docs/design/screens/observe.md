# Observe / 观 页面规格

本文是 Noema Observe / `观` 页的页面级规格。`观` 是进入一个 `境` 后查看全部照片的主视图，也是后续体验的入口。它不是整理进度页，不是 dashboard，也不是清理工具。

## 1. 页面职责

`观` 只做四件事：

1. 显示当前 `境` 的名称。
2. 查看此 `境` 内的全部照片。
3. 调整照片墙的排序和显示密度。
4. 进入后续体验：`甄`、`赏`、`鉴`。

`观` 不自动评分，也不承诺删除照片。`甄` 的可用性来自当前 workspace 已有的本地精筛结果；当 workspace 已有可复核组时，`观` 可以显示 `甄` 入口，但进入 `甄` 后才处理决策。

当前 Flutter 实现中，`观` 已承载真实 workspace 的主视图：Home 点击 `境` 后进入 `观`，Import 创建或追加照片后回到 `观`。照片墙使用真实宽高和真实时间，点击单张照片进入全屏照片查看器。

## 2. 不做什么

`观` 首版不允许出现：

- `正在整理`、进度条或分析 stage；
- 清理收益、节省空间、删除建议；
- AI chat、机器人或自动助手气泡；
- dashboard 统计卡片；
- 右上角更多菜单承载主要功能；
- `开始整理` 作为主动作；
- 系统照片删除入口。

## 3. 视觉基调

关键词：

```text
安静地看见这个境。
照片墙是主体。
体验入口在旁边等待，不抢夺观看。
```

视觉规则：

- 使用与 Home / `境`、Import / `入` 一致的手机画布宽度。
- 使用同一 `NoemaSceneFrame`、scene metrics、背景 painter 和 palette。
- 顶部 `Noema` wordmark 保持居中，不被工具按钮推偏。
- 背景大型低对比 `观` 主题字使用共享 `NoemaThemeMark`，不得页面级手调偏移。
- `Noema` 使用 `NoemaLatin`；`观`、境名、照片数量、底部一字功能标签以及所有中文 tooltip 使用 `Luo`。
- 工具图标优先；底部入口以一字命名为主，文字只保留境名、计数和极短标签。

## 4. 顶部结构

建议结构：

```text
返回图标              Noema

大型低对比「观」背景字

境名
照片数量        排序  导入  密度
```

顶部规则：

- 返回图标视觉盒左边界与主内容左边界对齐。
- `Noema` 的顶部坐标和视觉中心必须与 Home / `境`、Import / `入` 一致。
- 初版不放右上角更多菜单。
- 轻工具下沉到境名下方的信息行。

## 5. 照片墙

照片墙是 `观` 的主体。

首版采用稳定自然比例瀑布流。

原因：

- 同一密度下列宽稳定，排序改变顺序时，同一张照片不会因为同行组合变化而忽大忽小；
- 真实宽高能直接转成 tile 高度，更容易保持横图、竖图和近正方形图的可信比例；
- 大批量照片下重排更可控，能配合动画降级和可视区域构建减少卡顿。

规则：

- 横图、竖图、近正方形图都按自然比例显示。
- 密度决定列数和列宽，tile 高度由真实宽高比例决定；缺失或异常比例时使用安全 clamp。
- 行间距和列间距紧凑但不拥挤。
- 顶部可使用轻微渐隐，让照片墙融入标题区；底部参考 Home / `境` 使用短距离渐隐，避免硬切边缘，但不得形成大面积暗遮罩或压缩中间观看面积。
- 照片墙的绘制层需要保留左右阴影安全边距。窄屏 Android 上不得让最左 / 最右照片的阴影被 `ShaderMask`、滚动窗口或父容器裁掉。
- 单张照片不可用时使用克制占位，不影响其他照片展示。
- 如果已保存的缩略图 / 预览路径缺失或解码失败，但该照片仍有可读 `sourceUri`，照片墙应按单张照片尝试重建派生图并回写 workspace。不要因为长期未打开导致临时 cache 被清理，就让整个 `境` 长期显示灰色文件名占位。
- 不显示文件管理器式文件名列表。
- 100 张以上照片时保持虚拟化，只构建可见和近可见 tile；排序和密度切换不能让全部照片同时做重排位移动画。
- 点击单张照片进入全屏查看器；长按仍进入多选移除，不与点击查看冲突。

## 6. 照片墙工具

照片墙信息行包含：

- 照片数量，例如 `42 张`；
- 导入：继续添加照片；
- 查看选项：排序、筛选和照片墙密度。

这些工具尽量用图标，tooltip / 语义标签负责说明含义。不要把它们做成文字按钮。

`导入` 是主操作，继续外露在信息行中；排序、筛选和密度属于观看方式，收纳在同一个 `查看选项` 弹框。弹框横向分为三组，并使用弱竖线、组名和间距区分：

```text
排序        |  筛选        |  墙面
时间 / 评分 |  筛选循环按钮 |  密度循环按钮
```

排序语义：

```text
时间倒序：最新的照片在前
时间正序：最早的照片在前
评分高到低：已评分照片按总分从高到低，无分数照片排后并按时间兜底
评分低到高：已评分照片按总分从低到高，无分数照片排后并按时间兜底
```

时间排序使用导入时保存的真实拍摄时间；缺失时依次使用修改时间、导入批次时间和稳定 fallback。`观` 不提供名称排序和修改时间排序，因为照片通常不会在 Noema 内命名，也不会在 Noema 内修改原照片。

时间排序和评分排序互斥，只能有一个主排序。评分排序只在当前 `境` 至少有一张照片存在 AI 评分时显示。`全部 / 珍藏` 是筛选，不参与排序，但在 UI 上使用一个循环按钮切换状态；没有珍藏照片时筛选按钮置灰。照片墙默认不铺满评分数字；只有启用评分排序时，有分数的照片显示小分数角标。已珍藏照片始终可显示小实心爱心角标。

时间排序图标使用时间符号和独立方向箭头并排组合，方向箭头不能叠在日历格子、时钟轮廓或其他复杂图形内部，避免状态不可读。评分排序图标使用评分趋势符号和方向箭头。组合后的视觉重量要接近同排 24px 工具图标，箭头应小于主符号并紧贴其右侧。

## 7. 缩放与动效

用户可以通过双指缩放照片墙，也可以通过 `查看选项` 中的密度按钮切换。

首版使用三个离散级别：

```text
紧凑墙
标准墙
大图墙
```

交互规则：

- 双指放大吸附到更大的照片墙级别。
- 双指缩小吸附到更紧凑的照片墙级别。
- 密度按钮使用单个图标循环切换三个级别，不展开成三个并列按钮。
- 照片使用稳定 id 进行重排动画，位置和尺寸自然过渡。
- 动效借鉴 GSAP Flip：先记录旧布局，再过渡到新布局。
- 缓动借鉴 GSAP core 的 `power3.out` / `expo.out` 气质，Flutter 使用 `Cubic(0.16, 1, 0.3, 1)`。
- 大批量照片达到性能阈值时，重排位移动画降级为轻状态变化，优先保证滚动、排序和密度切换的响应性。
- reduced-motion 下减少位移感，但保留状态变化。

不要求任意比例无级缩放。美观、稳定和可控优先。

## 8. 全屏照片查看器

从 `观` 点击照片进入全屏照片查看器。它是 `赏` 方向的基础能力，但仍作为 `观` 内的单张照片查看体验；底部中间 `赏` 会进入独立的 `赏` Viewer，不展示 `鉴` sheet，也不继承 `观` 筛选。

规则：

- 初始照片应与用户在照片墙点击的照片一致，并沿用当前时间排序上下文。
- 横图、竖图和近正方形图都完整显示，不裁切。
- 页面可左右滑动浏览当前 `境` 的照片。
- 进入查看器后按需生成更大预览，最大边受控，不能为了查看器一次性加载全部原图。
- 双击照片在原尺寸视图和放大视图间切换，放大中心接近用户双击位置。
- 双指缩放最高不超过受控上限，放大后锁定横向翻页，避免缩放拖拽和翻页互相抢手势。
- 向内缩小到关闭阈值时可退出查看器，系统返回也应回到 `观`。
- 查看器背景保持极简，照片仍是主视角；允许复用 `鉴` 的 AI 品鉴 Sheet，但默认停在最低档，只露出少量底部 Sheet chrome，不主动遮挡照片。
- 预览不可用时只显示单张不可用状态，不影响其它照片查看。
- 若预览路径缺失但 `sourceUri` 仍可读，查看器应按需重新生成该单张预览；恢复失败才显示不可用状态。

## 9. 系统返回

Android 系统返回和手势返回必须符合移动端常规预期：

- 在 `观` 中返回到 Home / `境`，不直接退出应用。
- 在全屏照片查看器中返回到 `观`。
- 在从 `观` 进入的追加导入页中返回到 `观`。
- Home / `境` 才使用二次返回退出应用提示。
- 退出提示使用统一的 `NoemaHintBubble` 小胶囊样式，位于上半部视觉焦点区，固定中文文案为 `再返回一次，退出 Noema`，不带额外退出图标。

## 10. 底部功能入口

底部保留一个轻功能入口区。入口不做普通导航胶囊，也不做“图标 + 字”的工具按钮组合。`赏` 是可反复使用的沉浸入口，应作为视觉主动作；`甄` 与 `鉴` 更像阶段性任务入口，只在对应能力可用时于两侧低亮浮现，避免三者被理解成同级常驻导航：

```text
甄    赏    鉴
```

英文界面使用短标签，避免破坏签文入口比例：

```text
Cull    View    Rate
```

含义：

- `甄`：进入照片甄选体验，在其中进行相似、连拍或待确认照片的本地甄别与取舍；
- `赏`：进入全屏沉浸式照片欣赏 Viewer。
- `鉴`：鉴赏、评分或价值标记；

规则：

- 一字命名是主体，不再叠加同权重图标。
- 当前确认方案为 `dock=intent-seal`。它以一条可拖动 / 点按的底部意图轴承载三种意图，默认像吸附一样停在中间 `赏`，不拖动时自动复位到中轴。
- 中央活动签使用方形玻璃签语言，显示当前意图字。点击中间 `赏` 直接进入 `赏` Viewer；v1 从当前 `观` 排序列表第一张开始，不继承 `观` 的筛选状态。只有从中间 `赏` 起手并向上拖到当前可见照片 tile 后松手，才会从该照片开始进入 `赏` Viewer；从底部 `甄` / `鉴` 位置起手时，只保持底部意图轴选择，不转入照片墙命中。拖向可用的 `甄` 或 `鉴` 时，中签沿轴移动并替换为对应字，松手触发对应任务并回到中间。
- 手指直接落到 `甄` / `鉴` 区域时，中签应以清晰可见、顺滑的吸附动画移动到对应位置，不瞬间跳变；手指继续拖动时则保持跟手响应。
- 底部 `甄 / 赏 / 鉴` 意图轴及其上方缓冲区是拖赏取消区，不做照片 tile 命中；用户拖出 `赏` 后回到底部区域会清除照片高亮，松手取消本次拖赏。
- `赏` 的左右终点必须能与 `甄` / `鉴` 的视觉中心重叠，保证拖到最左 / 最右时字心和侧向隐章框心对齐。该对齐属于回归项，不应随意改动。
- 两侧 `甄` / `鉴` 只在可用时显示。可用态使用低亮字签、隐章角标、淡虚线和循环脉冲提示：脉冲从 `赏` 沿虚线向两侧移动，触达侧向图标后沿隐章框线扩散；不可用时两侧保持安静，不占据常驻按钮地位。
- 上方说明文字只在靠近可用侧向入口、或触发后短暂出现。文案保持短句，中文当前使用 `甄别相似`、`沉浸欣赏`、`留下判断`；说明应与对应目标中心对齐，位置比图标更高，出现时有轻微淡入和缩放，不用重提示气泡。
- `?dock=intent-seal&tune=1` 是调参预览入口，只暴露 `赏` 左 / 右终点数值输入，供手动微调拖动终点；其它对齐参数不作为日常调参入口。调参结果会导出到 localStorage 和 meta 元素的 `noema.intent_seal_tune.v1`，便于读取。
- 拖动 `赏` 指定起点只命中当前可见照片 tile，只高亮当前目标照片，不遍历整面照片墙，也不引入昂贵全墙视觉效果。
- 设计对照仍保留 `dock=lens`、`dock=object`、`dock=intent`、`dock=intent-ripple`、`dock=intent-tiles`、`dock=intent-rail`、`dock=intent-gate`、`dock=quiet`、`dock=balanced`、`dock=orbit`、`dock=rail`：它们只作为历史对照或后续实验入口，不再作为当前默认设计。
- 三个入口保持留白和低对比边界；不要把它们塞进厚重的整条底部卡片。
- 入口应像 Home / `境` 的底部创建入口一样轻覆盖在场景上，照片墙滚动窗口应延展到画布底部，让底部渐隐发生在更低的位置；同时必须保留足够滚动留白，避免最后一行被遮挡。
- 不叫 `辨`、`评` 或 `静观`。
- 不使用“开始整理”。
- 当前 `甄` 已进入 `ReviewGroupsScreen` 实装，支持快甄和对照甄。`赏` v1 进入单一沉浸 Viewer，底部控制层仅保留范围、顺序 / 随机、播放 / 暂停、播放间隔和横竖屏。`鉴` 已进入正式首版工作台，有照片时可进入；`观` 的照片放大页复用同一套 AI 品鉴 Sheet，已生成的品鉴结果随照片持久化。

### 10.1 甄入口可用性

`甄` 不应承担“开始扫描”的职责。用户进入 `甄` 时才检测相似组，会导致 `观` 无法判断是否有待处理内容，也无法做到按需浮现。当前实现采用导入 / 追加完成后的同步本地精筛结果来决定入口是否开放：

- Import 创建 `境` 或从 `观` 追加照片完成后，当前 workspace 会生成本地相似组、连拍组或待确认组。
- `观` 只根据当前已有可处理组显示 `甄` 入口；没有可处理组时保持安静，不把 `甄` 做成常驻同级导航。
- 用户点击 `甄` 后只进入处理流程，不应才开始首次扫描全部照片。
- Noema 不在应用启动时默认全图库后台筛选，也不承诺持续后台监听系统相册变化。

精筛信号分层：

- 轻量信号：导入批次、拍摄时间间隔、文件名序列、EXIF 时间、尺寸和媒体元数据。
- 视觉信号：缩略图 / 预览图生成后计算的多哈希、HSV 颜色、亮度布局和 SSIM 等本地特征。
- 时间同场景信号：在短时间窗口内，用结构、布局、颜色和亮度共同约束，补足人物移动、雾气 / 瀑布或轻微构图偏移造成的 SSIM 偏低。

触发与更新规则：

- 新建或追加照片后只比较当前 workspace 相关照片，并与当前 `境` 内已有照片做必要比较。
- 删除或移出照片后更新相关组状态。
- 识别结果存入当前 `境`，用于决定 `甄` 是否浮现、浮现强度和待处理数量。
- 如果缩略图或预览质量不足，尤其是 RAW / DNG 派生图异常，`观` 应保持保守，不为了显示入口而强行扩大相似组。

## 11. 境名编辑

`观` 允许修改当前 `境` 的名称，但编辑能力要轻，不打断照片浏览。

默认态：

- 只显示境名和一个低权重编辑图标，编辑图标放在标题行右侧，不贴在境名文字后面。
- 不显示 `0/10`。

编辑态：

- 境名原地变成输入框。
- 右侧显示保存和取消图标。
- 字数限制为 10 个字符，计数只在编辑态显示。
- 保存空名称时给轻提示，不保存空名。
- 不使用长下划线，避免切割背景 `观` 字。

## 12. 空状态

当前 `境` 没有照片时：

- 显示安静空状态；
- 提供添加照片入口；
- 不显示确认按钮；
- 不显示统计卡片；
- 不显示分析进度。

推荐文案：

```text
还没有照片
```

## 13. 多选移除

`观` 支持长按照片进入多选移除状态，交互与 `入` 的暂存区多选保持一致。

规则：

- 长按任意照片进入选择态。
- 选择态下点击照片切换选中。
- 选择态隐藏境名下方的照片墙工具行，改为显示取消和移除图标。
- 移除前弹出确认，说明 `原照片仍在系统相册中`。
- 确认后只从当前 `境` 移除照片，不删除系统相册原文件。
- 退出选择态后恢复照片数量、排序、导入和密度工具行。

## 14. COSS UI 参考

制作前检查 COSS UI 组件类别，适用结论：

- Toolbar：参考工具分组，不复制视觉。
- Toggle Group：参考密度切换和排序状态。
- Icon Button / Tooltip：图标按钮必须有语义标签。
- Scroll Area：参考滚动区域的边界和渐隐。
- Empty：参考空状态的单一动作结构。
- Toast / Notification：参考“顶部居中、轻量短提示”的职责；Flutter 实现不直接复用 COSS React 组件，统一使用 Noema 本地小胶囊提示。

Noema 不直接复制 COSS 视觉，而是借鉴组件职责、状态结构和交互分层。

## 15. 验收

初版 Web 预览必须确认：

- `观` 与 Home / `境`、Import / `入` 使用一致手机画布宽度。
- `Noema` 顶部居中，顶部高度和视觉中心一致。
- 背景 `观` 不裁切，不页面级偏移追齐。
- `观` 继承同一 light / dark / auto 色调来源。
- 境名、照片计数、排序、导入、密度工具布局清晰。
- 照片墙保留自然比例，布局紧凑但不杂乱。
- 密度切换时照片自然移动，不硬切跳格。
- 底部 `甄 / 赏 / 鉴` 使用 `intent-seal` 意图轴，不抢夺照片墙主体。
- 所有图标按钮有 tooltip / 语义标签。
- 点击照片进入全屏查看器，双击可放大 / 复位，放大后不误触横向翻页。
- 系统返回在 `观` 返回 Home，在照片查看器返回 `观`，只有 Home 二次返回退出。
