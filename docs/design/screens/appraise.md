# Appraise / 鉴 页面规格

本文是 Noema Appraise / `鉴` 页的页面级规格。`鉴` 已从早期“价值 / 理由 / 鉴语”原型进入正式首版工作台：它提供本机初见、分档浏览、珍藏标记、单张鉴赏 sheet、AI Provider 设置、单张 AI 品鉴、系列品鉴和结果持久化。

## 1. 页面职责

`鉴` 用于回答一个更慢的问题：

```text
这张照片在这个境里处于什么位置，值得怎样继续看？
```

首版职责：

- 按本机基础信号把当前 `境` 中照片分入 `微瑕 / 成片`；
- AI 品鉴完成后按 AI 总分允许照片进入 `佳作`；
- 当用户主观喜爱照片时，动态出现并持久化 `珍藏`；
- 用与 `观` 一致的真实比例照片墙承载浏览；
- 点击照片进入全屏查看器，并以底部 sheet 呈现 `初见 / 四维 / 总观 / 打磨 / 自问`；
- 在右上角保留 AI 设置入口位置，但不在 Provider 未配置前伪造 AI 结果。

## 2. 非目标

`鉴` 不应变成：

- 游戏化评分器；
- 文件管理标签页；
- AI 聊天面板；
- 照片编辑器；
- 系统相册删除入口；
- `甄` 的重复筛选页。

本机初见只能做基础可看性与结构判断，不宣称完成审美意义上的“佳作”判断。用户仍然是最终判断者。

## 3. 入口状态

- `鉴` route：`/appraise`。
- `观` 的底部意图轴在当前 `境` 有照片时开放 `鉴`。
- 从 `观` 点击单张照片仍进入普通全屏查看器；从 `鉴` 打开照片时，默认带出鉴赏 sheet。
- `/appraise-demo` 及 demo 资产不属于正式产品入口。

## 4. 分档规则

首版动态分档：

- `微瑕`：本机信号提示需要复核的照片，例如清晰度、曝光、主体重心或时间信息不足；
- `成片`：本机基础观看条件成立，适合继续留在当前境中；
- `佳作`：保留给未来 AI 品鉴或明确人工提升机制，本机基础评分不直接进入；
- `珍藏`：用户主观喜爱的标签，不覆盖系统分数含义。

AI 品鉴完成后，档位以 AI 总分为主：`80+` 进入 `佳作`，`60–79` 进入 `成片`，低于 `60` 留在 `微瑕`。本机清晰度、曝光等 hard / soft issue 只作为入档依据和风险提示展示，不再阻止高分照片晋级。

分数不默认外显。排序使用内部 score 进行高低切换，但 UI 只表达档位与判断内容。

## 5. 视觉规则

- 复用 `NoemaSceneFrame`、顶部 wordmark、`鉴` 背景主题字和 `NoemaGlassIconButton`。
- 顶部档位不是按钮卡片，而是半透明横线 + 向上渐变 + 文字的分类栏。
- 照片墙复用 `观` 的真实比例 masonry 逻辑，保留横竖图自然比例。
- 照片墙上下使用 `NoemaScrollEdgeFade` 渐隐，不硬切。
- 排序入口使用图标，不使用文字按钮。
- 右上角为空间工具位，当前用于 AI 设置入口。
- 中文使用 `Luo`，英文 wordmark 使用 `NoemaLatin`。

## 6. 交互规则

- 顶部最少显示 `微瑕 / 成片`，有对应内容时动态显示 `佳作 / 珍藏`。
- 点击档位切换照片墙内容。
- 点击爱心将照片加入或移出 `珍藏`，并随当前 `境` 持久化。
- 点击照片进入全屏查看器，左右滑动切图、双击放大、双指缩放和缩小退出复用 `观` 的照片查看器。
- `鉴` 中打开照片默认显示可拖动 bottom sheet；sheet 顶部保留拖拽把手。
- sheet 内容按 `初见 / 四维 / 总观 / 打磨 / 自问` 组织。
- `AI 品鉴 / 深度品鉴` 在 Provider 未配置前只提示设置，不上传照片，也不展示伪 AI 结论。

## 7. 后续要求

AI 与持久化已进入首版实现。后续继续补齐：

- 更完整的 Provider 能力白名单和错误说明；
- API Key 清除 / 重置入口；
- 自定义 Provider 的真实设备回归验证；
- 系列品鉴长任务的中断恢复和失败重试策略；
- 用户手动提升档位的持久化规则；
- `观` 中单张照片底部轻提示与完整 sheet 的联动；
- Android 模拟器或真机真实照片验证。

## 8. 验收

最低验收：

- `Noema`、背景 `鉴`、`微瑕 / 成片` 可见；
- 有照片时显示真实比例照片墙，不使用 demo 假图；
- 排序图标可切换高低顺序；
- 点击爱心后动态出现 `珍藏`；
- 点击照片进入全屏查看器，并显示鉴赏 sheet；
- sheet 上滑后能看到 `初见 / 四维 / 总观 / 打磨 / 自问`；
- 空 workspace 显示克制空态，不生成占位照片。
