# Appreciate / 赏 页面规格

本文是 Noema Appreciate / `赏` Viewer 的页面级规格。`赏` 是进入某个 `境` 后的单一沉浸式照片欣赏体验，不是整理模式、播放功能集合、分享页或 AI 赏语页。

## 1. 页面职责

`赏` v1 只做三件事：

1. 以全屏沉浸方式观看当前 `境` 的照片。
2. 在同一个 Viewer 中轻量控制播放范围、播放方式、播放状态、播放间隔和横竖屏。
3. 在退出时停止播放并恢复 Noema 主 App 的默认竖屏状态。

`赏` 不做照片评分、不做删除、不做分享、不做音乐、不做 AI 文案，也不展示 `鉴` 的 sheet。

## 2. 入口与退出

- 用户在 Observe / `观` 底部 `intent-seal` 中点击中间 `赏` 进入 Viewer。
- v1 从当前 `观` 排序列表的第一张照片开始。
- v1 不继承 `观` 的筛选状态；例如 `观` 当前只看珍藏时，`赏` 仍按自己的播放范围默认全选。
- 进入 Viewer 时使用慢速黑场淡出过渡，让体验像影片开场一样自然沉入照片。
- 点击左上角返回、系统返回或 route pop 都必须退出 Viewer 并回到 `观`。
- 退出时必须停止自动播放，并请求恢复竖屏。

Observe 底部 `赏` 支持拖到当前可见照片 tile 后从该照片开始；v1 不做停留确认，只做可见 tile 命中、单 tile 反馈和松手进入。照片墙指定起点必须从中间 `赏` 起手；从底部 `甄` / `鉴` 位置起手时，不转成照片墙命中。

## 3. 视觉与 VI

关键词：

```text
安静看照片。
控制层短暂出现。
照片本身承担情绪。
```

规则：

- 左上角只保留返回入口，不在 Viewer 内额外显示独立 `赏` 水印。
- `赏` 的 VI 识别由 `观` 底部入口和路由语义承载；Viewer 内优先保证照片观感，不把 `赏` 放到右上角或覆盖照片。
- 前景照片完整显示，使用 contain，不允许裁切。
- 非照片区域使用当前照片的模糊放大背景补足。
- 顶部和底部使用黑色渐变遮罩，保证浮层可读。
- 底部控制条为半透明黑色胶囊，轻微 blur，浮在照片上。
- 不新增装饰性渐变、光球或 AI 风格色彩。

## 4. 照片展示

- 横图、竖图和近正方形图都必须完整展示。
- Viewer 按需使用大预览，不一次性加载全组原图。
- 当前页必须有界预热前后邻近照片的 preview，并按当前显示尺寸预解码，避免切到未看过照片时出现黑屏或空白帧。
- 程序化播放切换前应先预热目标照片；播放节奏可以稍慢，但不应让动画先于目标图可显示。
- 不预加载全量 preview，不预生成整组模糊背景。
- 预览不可用时只显示当前照片的不可用状态，不影响其它照片。

## 5. 基础交互

- 左右滑动切换照片。
- UI 显示态下单击屏幕隐藏控制层。
- UI 隐藏态下单击屏幕恢复控制层。
- UI 显示态下如果 3 秒没有任何操作，自动隐藏控制层进入沉浸欣赏。
- UI 显示态保留现有双击放大 / 复位和双指缩放能力。
- UI 隐藏态禁止双击 / 双指缩放；隐藏态的双击或双指动作只唤起 UI，并设置短暂防误触窗口，避免连续 tap 立即触发缩放。
- 放大后仍沿用现有 Viewer 的横向翻页锁定规则，避免缩放拖拽和翻页抢手势。

## 6. 底部控制层

底部控制层是唯一控制区，不做更多菜单。控制层包含五项：

1. 播放范围。
2. 顺序 / 随机。
3. 播放 / 暂停。
4. 播放间隔。
5. 横屏 / 竖屏。

控制条上方使用复用 `甄` 页面风格的无数字页位指示器，只在 UI 显示态顺带显示；不显示 `3 / 24` 这类数字页码。控制文本必须短，避免在手机宽度内挤压或换行失控。

## 7. 播放范围

支持四个范围：

```text
微瑕
成片
佳作
珍藏
```

规则：

- 默认四类全选，摘要显示 `全部`。
- 用户可以多选。
- 至少保留一个分类，并且切换后的播放列表必须至少有一张照片。
- 如果切换会得到空列表，保持上一次选择。
- `微瑕 / 成片 / 佳作` 完全复用 `鉴` 的分档逻辑。
- `珍藏` 是用户主观标签，不覆盖照片的 `微瑕 / 成片 / 佳作` 分档。
- 一张照片命中多个已选分类时只出现一次。
- 如果当前照片被筛选剔除，跳到筛选后列表的第一张。

摘要规则：

- 四类都选中：`全部`
- 选中一个：对应名称，例如 `佳作`
- 选中两个：用 `+` 连接，例如 `佳作+珍藏`
- 选中三个：`3类`

## 8. 播放方式

- 支持 `顺序` 和 `随机`。
- 顺序播放遵循当前 `观` 排序。
- 随机播放使用洗牌队列，避免短时间重复。
- 播放方式只是控制项，不是独立页面或模式。
- 用户手动滑动后，随机队列应围绕新的当前照片重新同步。

## 9. 自动播放与间隔

- 进入 Viewer 默认暂停。
- 点击播放后按当前范围、方式和间隔自动切换照片。
- 点击播放后立即隐藏 UI，进入无控制层的沉浸播放态。
- 自动播放切换照片使用较慢、柔和的 dissolve-style 淡入淡出节奏，避免硬切、快速跳动或明显横向生硬滑动。
- 自动播放切到下一张前必须走 Viewer 的邻近照片预热链路，不能在目标图还未生成 / 解码时先显示黑屏。
- 播放中用户仍可左右滑动。
- 当前播放列表只有 1 张时，播放按钮置灰或不可用。
- 播放间隔只支持 `5s / 10s / 30s`，默认 `10s`。
- 点击当前间隔值后弹出轻量选择，不做自定义秒数。
- `5s / 10s / 30s` 使用现有中文正文字体，不使用独立数字字体。

## 10. 横竖屏

- Noema 主 App 默认竖屏，不跟随系统自动横屏。
- `赏` Viewer 进入时默认请求竖屏。
- 控制层的方向按钮显示当前状态：竖屏时显示 `竖屏`，横屏时显示 `横屏`。
- 竖屏状态下点击方向按钮后，仅 Viewer 请求横屏显示，按钮变为 `横屏`。
- 横屏状态下点击方向按钮后，Viewer 请求回到竖屏，按钮变为 `竖屏`。
- 横屏后左上角仍然是返回入口，底部控制层仍位于视觉底部。
- 退出 Viewer 时必须恢复竖屏。

## 11. 不做什么

v1 不做：

- 分享
- AI 赏语
- 背景音乐
- 更多菜单
- 查看鉴 Sheet
- 多种沉浸模式
- 播放速度自定义
- 复杂策展顺序
- 独立随机模式页面

## 12. 拖动起点

- Observe 底部 `赏` 支持拖动到当前可见照片 tile 后松手进入 Viewer。
- 只有从中间 `赏` 起手才允许进入照片墙命中；从底部 `甄` / `鉴` 起手时，手势仍属于底部意图轴。
- 拖动进入照片墙时，只命中当前可见 tile，不遍历非可见照片。
- 命中反馈只作用于当前目标照片，使用轻量高亮、边缘亮度和轻 scale，不做全墙滤镜或昂贵 shader。
- 如果拖动没有命中照片，不进入 Viewer，也不误触发其它功能；点击 `赏` 仍按普通入口进入。
