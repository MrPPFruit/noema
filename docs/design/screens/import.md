# Import / 入 页面规格

本文是 Noema Import / `入` 页的页面级规格。`入` 有创建态和追加态：从 Home 进入时创建一个新 `境`，从 `观` 的添加照片进入时把新照片添入当前 `境`。`入` 不负责整理、分析、判断或删除系统照片。

## 1. 页面职责

创建态的 `入` 只做三件事：

1. 填写一个 `境` 的名称。
2. 多次选择照片，把照片追加到这个 `境` 的暂存区。
3. 在名称和照片都满足条件后完成创建。

`入` 完成后生成一个 `境`。后续 `观` 是进入这个 `境` 后查看全部照片与进入后续体验的主视图；不要在 `入` 中强制进入 `观`、`甄` 或任何整理流程。

追加态的 `入` 只显示当前 `境` 的名称和已有照片数量，不再显示空的命名输入。追加态的缩略图区只展示本次准备添入的照片，不展示当前 `境` 已有的全部照片；已有照片的查看仍由 `观` 负责。

## 2. 不做什么

`入` 不允许出现：

- `开始整理` 作为主动作；
- AI 分析承诺；
- 清理收益、节省空间、删除建议；
- dashboard、统计卡片、任务列表；
- 文件管理器式工具栏；
- 常驻信任 copy，例如“这些分析在本机完成”“不会改动你的相册”。

信任和本机分析说明属于 `观` 或更具体的功能上下文，因为用户进入 `观` 后才真正关心这个 `境` 的照片如何被查看、分析或进入后续体验。

## 3. 视觉基调

关键词：

```text
安静的入口。
创造一个境。
照片进入记忆空间前的暂存台。
```

视觉规则：

- 使用与 Home / `境` 一致的手机画布宽度和页面骨架，不在 Web 预览中横向铺满浏览器。
- 使用与 Home / `境` 一致的 scene metrics、顶部高度、背景 painter 和色调 palette；不能在 `入` 页单独硬编码背景色、安全区偏移或顶部坐标。
- 背景保留大型低对比 `入` 主题字，继承 Home / `境` 的空间语言。
- `入` 主题字必须使用共享 `NoemaThemeMark`，并与 Home / `境` 的主题字保持统一视觉锚点；禁止在页面内临时挪动到导致裁切的位置。
- 照片出现后，缩略图成为页面视觉主体。
- 控件保持图标优先，除输入框、计数和极短目的说明外，不堆文字。
- 不使用明显卡片套卡片，不把页面做成文件管理器。

## 4. 顶部结构

建议结构：

```text
返回图标              Noema              选择态工具

大型低对比「入」背景字
```

顶部动作：

- 返回：图标按钮。
- 返回按钮的玻璃视觉盒左边界必须与页面主内容左边界对齐；点击热区可以略微外扩，但视觉盒不能比境名输入列更靠右。
- `Noema` 默认居中，保持与 Home / `境` 一致。
- `Noema` 的顶部坐标和视觉中心必须与 Home / `境` 一致；后续页面也应沿用同一规则。
- 完成创建：底部右侧方形玻璃勾号按钮。
- 完成按钮仅在至少导入 1 张照片后出现；境名为空时按钮可见但低亮，点击后在勾号上方浮出非阻断命名提示。
- 境名为空但已有照片时，完成按钮属于“低亮可点”状态：它应比可用态更安静，但在浅色和深色模式中都必须看得清图标、边框和方形轮廓。
- 图标必须有语义标签和 tooltip。

## 5. 境名

规则：

- 境名必填。
- 不自动生成默认名。
- 不使用 `未命名境`。
- 最多 10 个汉字字符感知长度。
- 前后空格自动裁剪。
- 只输入空格视为空。
- 第一版允许重名，不把命名做成文件系统约束。

输入表现：

```text
为境命名
0/10
```

字数计数放在输入区左侧下方，避免与照片计数在右侧堆叠。输入区不使用横贯页面的下划线，避免切割背景 `入` 主题字；依靠大号 Luo 占位文字、左侧计数和占位文字右侧的轻量闪烁输入光标表达输入状态。

英文界面中的境名占位与输入文本使用 `NoemaLatin`，中文界面使用 `Luo`，并需要保证占位文字与光标在窄屏内不溢出。

界面不显示长篇校验说明。有照片但未命名时，完成图标置灰；用户点击后在勾号上方短暂浮出轻提示：

```text
先为境命名
```

提示自动消失，不显示遮罩或关闭按钮，同时焦点回到境名输入区。动效参考 GSAP core 的 `autoAlpha`、`y` 和 `scale` 组合：从勾号方向轻微上浮、淡入并缩放到位，避免动画 layout 属性。

## 6. 导入照片

第一版支持：

- 多次从系统照片选择器选择照片；
- 每次选择后回到 `入`；
- 新照片追加到当前 `境` 暂存区；
- 不显示“使用样例照片”；
- 不承诺文件夹递归导入。

Android 端导入遵循轻量媒体管线：

- 第一次进入导入并点击添加照片时，先申请图库读取权限。权限用途是 MediaStore 索引、缩略图预热、派生图恢复和本地视觉分析，不是在应用启动时默认全图库后台扫描。
- 权限可读后再打开系统 Photo Picker。用户面对的选择界面必须继续是系统 Photo Picker 或系统内容选择 fallback，不使用自研图库浏览器替代。
- 打开 Android Photo Picker 时请求本地数据优先，用于降低云端照片尚未本地可读时触发系统“部分照片无法加载 / 稍后再试”弹窗的概率。该弹窗属于系统 picker 内部行为，Noema 不能保证完全消除。
- 如果图库读取权限被拒绝，`入` 不打开 Photo Picker，也不创建部分 workspace；页面只显示克制的权限必需状态或轻提示。
- 系统选择器返回后，Noema 优先保存 `content://` 引用、文件名、真实宽高、拍摄 / 修改时间、MIME 和文件大小，不在导入阶段复制全部原图。
- 缩略图按显示需要以小并发后台生成并逐张进入网格；单张生成失败只影响对应 tile，不阻塞整批照片出现。
- Android 缩略图生成不能在主线程同步执行解码、压缩和写盘；应使用有边界的后台 worker、复用当前版本 cache，并把导入网格缩略图控制在接近 tile 尺寸的上限。
- Android 缩略图、预览和图库索引派生文件写入应用文件目录 `noema_media/`，不能只放临时 cache。长期未打开后如果派生路径缺失或失效，应按单张照片从 `sourceUri` 尝试重建，而不是让整批照片停留在灰色文件名占位。
- Android 端不能把系统方形 `loadThumbnail` 结果当作最终缩略图来源；应按原图比例采样，读取 EXIF orientation，并在 EXIF 缺失时参考 MediaStore orientation 后再旋正/翻转，避免横图被竖排或缩略图自带黑边。
- 点击预览时再为单张照片生成较大预览图，不能为了导入列表提前读取或解码全部原图。
- 导入中的状态文案使用 `正在打开相册` 或 `正在带入照片`，避免长时间停在静态的 `正在准备照片`。
- Android 正式导入路径在缩略图生成后，需要读取缩略图 bytes 参与本地视觉分析。这样复制同一张真实照片时可以进入 `nearDuplicate` 甄别组，而不是只依赖拍摄时间或文件名兜底。
- 创建 / 追加确认按钮必须等元数据和缩略图分析准备结束后再可完成，避免用户在视觉 hash 尚未写入前创建 workspace，导致相似图识别缺失。

后续可单独设计：

- Android 文件夹导入；
- 相册 / collection 选择；
- 包含子文件夹扫描；
- 大批量导入上限和分批确认。

## 7. 空状态

空状态只表达当前动作结果，不做信任解释。

推荐文案：

```text
让照片入境
```

主入口使用与 Home / `境` 创建入口一致的底部方形玻璃按钮。图标使用“添加到照片”的语义，tooltip / 语义标签为 `添加照片`，不把入口放入缩略图第一格。

空照片状态不显示右侧完成按钮，避免用户把“确认空境”误认为可用路径。

## 8. 缩略图网格

导入后以 4 列缩略图展示。

规则：

- 每行 4 个。
- 可上下滚动。
- 缩略图稳定为正方形或近正方形。
- 网格只展示用户导入的照片，不混入导入入口。
- 继续添加照片入口固定在底部居中，位置与 Home / `境` 创建入口一致。
- 显示轻量计数，例如 `24 张`。
- 已导入照片优先显示真实缩略图；若图片不可用或加载失败，显示克制的不可用占位，不让整批失败。
- Android 原生端使用 Noema 媒体桥生成有边界且已按 EXIF / MediaStore 方向校正的缩略图；iOS / Web 保持现有 picker 路径。所有缩略图显示都需要使用接近实际 tile 尺寸的 cache hints，避免 100 张以上照片时解码过大图片。Android 导入网格缩略图默认上限为 320px，避免为 4 列小 tile 生成过大的 512px 文件。

## 9. 预览

点击缩略图打开放大预览。

预览规则：

- 全屏或近全屏查看；
- 预览必须完整显示照片，不裁切；横图按宽度完整显示，竖图按高度完整显示；
- 关闭按钮使用紧凑方形玻璃按钮，放在图片下方，并与图片保持明确间距；
- 关闭按钮在 light / dark 预览遮罩上都必须保持高对比度；浅色预览不得复用低透明轻玻璃按钮。
- 不显示编辑、评分、删除系统照片等动作；
- 如果图片不可用，显示不可用状态。

## 10. 多选与移除

长按缩略图进入多选模式。

多选模式：

- 长按的图片立即选中。
- 点击缩略图切换选中状态。
- 顶部品牌锚点保持稳定，不在 `Noema` 旁拼接已选数量。
- 暂时隐藏目的文案和照片计数，在同一行替换为取消选择和删除两个图标。
- 取消和删除入口靠近缩略图区域，而不是挤在最顶部。
- 删除图标触发确认，不直接删除。

删除确认文案：

```text
从此境移除
原照片仍在系统相册中
```

这里的删除只从本次导入暂存区移除，不触碰系统相册。

删除确认必须适配 Noema 手机画布。即使在 Flutter Web 的宽浏览器视口中，确认框也不能按浏览器宽度铺开，最大宽度应控制在手机画布内，例如 `390 - 48 = 342`。

## 11. 完成创建

完成创建条件：

1. 境名非空；
2. 至少导入 1 张照片。

满足条件后，底部右侧勾号可用。点击后创建一个 `境`。如果已经有照片但境名为空，点击勾号只浮出命名轻提示，不创建 `境`。

当前已有 `观` 主视图后，创建成功应进入 `观`，让用户查看这个 `境` 内的全部照片和后续体验入口。这里仍然不能自动进入 `甄`、分析、整理或决策流程。

## 12. 追加照片

从 `观` 点击添加照片进入 `入` 时，页面进入追加态：

- 不显示 `为境命名` 输入框。
- 显示当前境名，例如 `友人`。
- 显示已有数量，例如 `18 张已在此境`。
- 目的文案使用 `添入此境`。
- 右侧计数显示本次新增数量，例如 `本次 6 张`。
- 缩略图只展示本次新选择的照片，支持预览、长按多选和从本次追加中移除。
- 勾号确认后追加到当前 `境` 并回到 `观`。
- 与当前 `境` 已有照片重复的选择应跳过，不进入本次缩略图区。

## 13. 动效

### Home / 境 → Import / 入

动效参考：GSAP Flip 的共享布局状态过渡。Flutter 实现不引入 GSAP，而是借鉴其动效原则：

- 旧状态：Home 底部 `+` 创建动作和 `境` 空间。
- 新状态：Import 页面里的添加照片入口和 `入` 空间。
- 过渡：轻微缩放、淡入、位移和背景主题字交替。
- 目标：让用户感觉是从 `境` 的入口进入 `入`，而不是硬切页面。

Flutter 初版建议：

- `PageRouteBuilder` 控制页面淡入和平移；
- `Hero` 或共享 tag 用于创建入口的轻微形变；
- `入` 背景字慢一拍出现；
- reduced-motion 下保留状态变化，降低位移和缩放。

### 页面内部

- 添加照片后缩略图以轻微淡入出现。
- 长按进入多选时，只显示选择描边和工具图标，不做剧烈动效。
- 删除确认使用底部轻浮层或克制 dialog。

## 14. COSS UI 参考

制作前必须检查 COSS UI 组件类别，并记录适用结论：

- Field / Input：境名输入、计数、必填状态。
- Empty：无照片状态。
- Scroll Area：4 列缩略图滚动。
- Dialog / Drawer：删除确认和图片预览容器。
- Icon Button：返回、添加、完成、删除、取消选择。

Noema 不直接复制 COSS 视觉，而是借鉴组件职责、状态结构和交互分层。

## 15. 验收

初版 Web 预览必须确认：

- `入` 是全屏页面；
- `入` 与 Home / `境` 使用一致手机画布宽度；
- `Noema` 顶部居中；
- `Noema` 顶部高度、视觉中心和 Home / `境` 对齐；
- 背景 `入` 主题字与 Home / `境` 的背景主题字保持统一视觉锚点；
- 背景 `入` 不被裁切，不使用页面级偏移去追齐；
- `入` 继承 Home / `境` 的 light / dark / auto 色调来源；
- 境名必填；
- 境名为空时，`为境命名` 右侧显示轻量闪烁输入光标；
- 无“使用样例照片”正式入口；
- 无“开始整理”主动作；
- 可多次添加照片；
- 缩略图为 4 列；
- 点击可预览；
- 长按可多选；
- 删除确认说明不会删除系统相册原照片；
- 无照片时不显示完成按钮；有照片但无境名时完成按钮低亮，点击在勾号上方显示非阻断命名轻提示；满足“境名 + 至少 1 张照片”后通过底部右侧方形勾号完成；
- 添加照片入口、完成入口、预览关闭入口和 Home 创建入口使用同一个方形玻璃按钮组件语言；
- 多选时隐藏目的文案和照片计数，改显示取消与删除图标；
- 导入入口不占用缩略图第一格；
- 已导入照片优先显示真实缩略图，只有不可用或加载失败时才显示文件名占位；
- 点击预览时横图和竖图都完整显示，关闭按钮位于图片下方；
- 浅色和深色预览中关闭按钮都清晰可见；
- Home → Import 有轻量过渡动画。
