# Cull / 甄 页面规格

本文是 Noema Cull / `甄` 页的页面级规格。`甄` 用于处理已经发现的相似、连拍或待确认照片组，让用户在本地、可逆、克制的界面里决定保留或移出当前 `境`。

## 1. 页面职责

`甄` 只做三件事：

1. 展示当前 `境` 中可复核的相似组或连拍组。
2. 提供快甄和对照甄两种处理方式。
3. 把用户决策写回当前 workspace 的本地决策状态。

`甄` 不直接删除系统相册原照片。`清除出境` 只删除 Noema 本地 workspace 中被标记出境的照片记录和决策，不触碰系统相册。

## 2. 入口与状态

- 从 `观` 的 `甄` 入口进入。
- `观` 中只有当前 workspace 已有可处理的本地精筛组时，`甄` 才作为默认可用入口浮现。
- 实验 / 调参模式下可以强制显示入口，但正式体验不应把不可用的 `甄` 做成常驻主按钮。
- 没有可复核组时，显示安静空状态：`当前没有可复核的相似组。`

导入或追加完成后的本地精筛结果负责决定是否有可复核组；`甄` 是处理入口，不是扫描入口。正式体验不应在用户点击 `甄` 后才开始首次扫描，也不应在应用启动时默认全图库后台筛选。

### 2.1 识别与合并质量边界

- 近重复识别优先使用多种本地视觉签名：感知哈希、wavelet hash、HSV 颜色直方图、亮度布局和 SSIM 结构复核。
- 初始强相似分组使用 complete-linkage，避免链式弱相似把不同场景桥接进同一组。
- 短时间同场景合并只在拍摄时间接近、亮度变化可控、结构 / 布局 / 颜色同时过安全线时生效；它用于处理人物移动、瀑布 / 雾气变化和轻微构图偏移，不是全局降阈值。
- 缺少可靠拍摄时间时，只允许同批相邻导入且视觉信号更强的照片参与估算时间合并。
- RAW / DNG 识别依赖平台提供可代表原图内容的缩略图或预览；如果预览质量异常，`甄` 应保持保守，不为了召回率强行跨组硬合并。

## 3. 顶部与视觉

- 顶部使用与 Home / `境`、Import / `入`、Observe / `观` 一致的顶部品牌形式。
- 返回图标位于左侧，`Noema` wordmark 顶部居中。
- 背景主题字使用共享 `NoemaThemeMark`，内容为 `甄`。
- 页面不使用厚重黑色底图或额外卡片套卡片；照片、弧线和缩略图应直接融入 Noema 场景。
- 中文展示文字和一字标签使用 `Luo`。

## 4. 相似组列表

相似组列表用于进入具体处理：

- 每个组显示代表照片、组内数量和轻量状态。
- 新组不显示默认 `0%` 进度，避免用户以为系统已经替他判断完成度。
- 进入详情后可以选择快甄或对照甄。

## 5. 快甄

快甄适合单张连续处理。

布局规则：

- 当前照片位于中心区域。
- 上方为 `丢弃 / Discard` 区域，下方为 `保留 / Keep` 区域。
- 上下文字与对应弧线的距离必须一致，保持垂直对称。
- 中间照片到上下区域的视觉间距应均衡，不让底部缩略图挤压 `保留` 标签。

交互规则：

- 向上拖动当前照片，记录为 `Review for removal`。
- 向下拖动当前照片，记录为 `Keep`。
- 决策后照片从中心移出，对应队列出现缩略图。
- 队列是横向缩略图条，一行约 5 张，更多可横滑。
- 新加入缩略图从最左侧进入，旧缩略图向右移动一格。
- 点击任意缩略图：从对应队列移除，清除原决策，把这张照片召回中心重新评估。
- 如果召回时中心还有未处理照片，当前照片先回到待处理堆顶，召回照片成为当前卡。

动画规则：

- 当前照片拖到对应区域后应沿拖动方向自然消失。
- 缩略图出现应有轻微位移、淡入和队列重排，不做强烈弹跳。
- 动效可借鉴 GSAP Flip 的状态到状态过渡，但 Flutter 实现不引入 GSAP 依赖。

## 6. 对照甄

对照甄适合两张照片并排比较。

布局规则：

- 竖屏中左右两张照片并排出现，降低学习成本，复用快甄的上下 `丢弃 / 保留` 方向。
- 左右照片应保留自然比例和足够间距，不能重叠、拥挤或被顶部 / 底部 chrome 遮挡。
- 当只剩最后一张照片时，取消空线框，按快甄单图逻辑把最后一张照片居中放大展示。

交互规则：

- 用户可以拖动任一照片到上方或下方区域记录决策。
- 被处理照片离开后，下一张候选补入对应位置。
- 点击已进入队列的缩略图可撤销决策并召回该照片。
- 决策写回 workspace，与快甄共用同一套决策模型。

## 7. 大图预览

单图预览：

- 快甄和对照甄最后一张单图预览应与 `观` 的照片查看方式保持一致：完整显示照片，不裁切。
- 不显示额外关闭按钮；退出方式是系统返回或点击图片外区域。
- 支持双击放大、双指缩放、缩小退出和系统返回关闭。

双图预览：

- 对照甄两张照片都存在时，点击任一照片进入成对同步预览。
- 两张照片上下排列，使用同一缩放 / 平移交互状态，便于同步查看细节。
- 打开时左右两张图分别沿轨迹移动到预览位置，帮助用户理解对应关系。
- 可以使用很克制的照片边缘描边帮助区分左右来源；描边贴近照片边缘，放大超出边界时不强行显示外框。
- 不使用黑色底图、重卡片、重置按钮或关闭按钮。
- 退出方式是系统返回或点击图片外区域。

## 8. 清除出境

`清除出境` 只处理已经被标记为 `Review for removal` 的照片：

- 点击后弹出确认。
- 确认文案必须说明它删除的是 Noema 本地数据，不删除系统相册原照片。
- 确认后从当前 workspace 移除对应照片，并清除相关决策。

## 9. 验收

自动化测试至少覆盖：

- 空状态；
- 决策写回 workspace；
- 快甄上 / 下拖动决策；
- 快甄缩略图召回和队列左插入；
- 快甄上下标签与弧线间距对称；
- 缩略图为正方形且不遮挡标签；
- 对照甄拖动、召回和最后单图居中；
- 对照甄成对同步大图预览；
- 系统返回和点击图外关闭预览。

发布前还应在 Android 模拟器使用真实照片验证相似组识别和渲染，不只依赖占位图。
