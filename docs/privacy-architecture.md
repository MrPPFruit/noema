# 隐私架构

状态：当前公开仓库透明文档。

## 数据流

```text
用户主动选择照片
  -> 本机读取照片元数据和预览
  -> 本机建立整理空间、相似组、评审结果和浏览状态
  -> 用户可选配置 AI Provider
  -> 用户主动触发单张或系列 AI 品鉴
  -> 请求发往用户选择的 Provider
  -> 品鉴结果回写到本机工作流
```

## 默认本机行为

- Noema 不默认后台扫描全图库。
- Noema 不自动删除原始照片。
- Noema 不默认上传照片。
- Noema 不要求用户创建账号。
- Noema 当前没有自建账号、云同步或中转服务。

## 本机保存内容

每次公开 release 前需要逐项验证并写实：

- 工作空间和导入状态。
- 分组、评分、珍藏和 AI 品鉴结果。
- 缩略图、预览缓存和临时文件。
- AI Provider 设置。

## AI 请求边界

只有用户启用 AI 品鉴、配置 Provider 与 API Key，并主动触发品鉴时，相关照片数据才会发送给用户选择的 Provider。

## 发布前检查

- Android 权限与实际功能一致。
- 无默认后台请求。
- 日志不包含 API Key、照片、prompt、Provider 响应全文或本地敏感路径。
- release notes 标明隐私和网络行为是否变化。
