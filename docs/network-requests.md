# 网络请求

状态：当前公开仓库透明文档。

## 原则

默认照片整理不需要网络请求。联网行为应当由用户动作触发，并且能够解释目的地、是否包含照片、是否包含 API Key。

## 预期请求清单

| 功能 | 目的地 | 是否包含 API Key | 是否包含照片 | 连接方式 | 触发方式 |
|---|---|---:|---:|---|---|
| AI 单张品鉴 | 用户配置的 Provider | 是 | 是 | App 直接请求用户配置的 Provider | 用户主动触发 |
| AI 分类批量品鉴 | 用户配置的 Provider | 是 | 是 | App 直接请求用户配置的 Provider | 用户主动触发 |
| AI 系列品鉴 | 用户配置的 Provider | 是 | 是 | App 直接请求用户配置的 Provider | 用户主动触发 |
| 千问 API Key 帮助链接 | 阿里云帮助页 | 否 | 否 | 用户点击后打开外部网页 | 用户点击帮助 |
| GitHub Release 下载 | GitHub | 否 | 否 | 用户主动访问 GitHub | 用户下载 |

## 发布前验证

每次公开 release 前需要通过运行日志、代理或平台工具确认：

- App 启动时没有意外后台请求。
- 默认照片整理不发起网络请求。
- 未配置 AI Provider 时不能发起 AI 请求。
- 请求错误不会把 API Key 写入日志或 UI。
