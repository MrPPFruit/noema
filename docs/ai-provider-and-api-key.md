# AI Provider 与 API Key

状态：当前公开仓库透明文档。

## Provider 配置

Noema 的 `鉴` 支持用户配置自己的 AI Provider。Provider、Base URL、Model 和 API Key 由用户输入，Noema 不提供内置密钥。

## API Key 存储

- API Key 保存在设备安全存储中。
- UI 不回显已保存密钥。
- 重新输入会替换旧密钥。
- 公开仓库不得包含默认密钥、测试密钥或真实请求样例中的密钥。

## 请求触发条件

AI 请求只在用户主动触发以下动作时发生：

- 单张照片 AI 品鉴。
- 当前分类批量 AI 品鉴。
- 系列照片 AI 品鉴。

## 请求内容

请求可能包含：

- 用户配置的 API Key。
- 用户选择的模型参数。
- 需要品鉴的照片数据。
- Noema 生成的摄影品鉴 prompt。

请求由 App 直接发往用户配置的 Provider。Noema 当前没有自建账号、云同步或中转服务。

## 清除与替换

当前 UI 支持重新输入并替换旧密钥。后续 release 如果增加显式清除按钮，需要同步更新本页和隐私说明；在此之前，公开文档不能承诺“可清除”。
