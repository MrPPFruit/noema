# 安全政策

状态：当前公开仓库说明。

## 报告安全问题

如果你发现 Noema 的安全或隐私问题，请不要先公开披露。请优先通过 GitHub Security Advisory 提交；如果入口不可用，可以先创建不含敏感细节的 issue，请求维护者建立私下沟通。

## 重点关注范围

- API Key 泄露。
- 日志、错误提示、截图中包含密钥、照片、prompt 或本地路径。
- 意外后台网络请求。
- 非用户主动触发的照片上传。
- Android 权限扩大。
- release 包被替换、签名不一致或校验信息缺失。

## 非安全问题

以下问题一般走普通 issue：

- UI 排版问题。
- 文案错误。
- 性能优化建议。
- 功能请求。

## 响应边界

Noema 是独立开源项目。报告安全问题或提交贡献不构成固定响应时限、定制开发或长期维护承诺。
