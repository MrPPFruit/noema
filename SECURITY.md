# 安全政策

状态：当前公开仓库说明。

## 报告安全问题

如果你发现 Noema 的安全或隐私问题，请不要先公开披露。请通过 GitHub Security Advisory 提交，或使用仓库 README 中列出的安全联系人。

在正式公开前，安全联系人需要由项目负责人确认。

## 重点关注范围

- API Key 泄露。
- 日志、错误提示、截图中包含密钥、照片、prompt 或本地路径。
- 意外后台网络请求。
- 非用户主动触发的照片上传。
- Android 权限扩大。
- release 包被替换、签名不一致或校验信息缺失。

## 非安全问题

以下问题一般走普通 issue：

- UI 排版问题。
- 文案错误。
- 性能优化建议。
- 功能请求。

## 响应边界

Noema 是独立开源项目。Sponsor 或贡献不构成商业 SLA、定制开发或固定响应时限承诺。
